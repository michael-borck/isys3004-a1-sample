## What a real thinking chat covers

A Week 5 thinking chat would explore:

- What APIs are free, reliable, and don't require an API key (important for a public portfolio)
- What JSON is and what a typical API response looks like
- What `async/await` is and why you need it (vs why you can't just use a regular function)
- What can go wrong with an API fetch and how to handle it
- Why `response.ok` needs to be checked separately from the fetch itself
- What a loading state is and why it matters for user experience

## Prompt examples from this week

> "I need a free API for my portfolio that doesn't require a key, is reliable, and returns simple JSON I can display on a page. I want it to feel like a real feature, not just a technical exercise. What are my options?"

> "Before I write any code, explain what async/await does in plain English. Why can't I just call fetch() like a normal function?"

> "What can go wrong when fetching from an API? Walk me through every failure scenario and what I should do about each one."
