## What a real build chat covers

A build chat for Week 2 would show:

- A specific opening prompt describing exactly what you need (because you already thought it through)
- The AI's initial HTML output
- Follow-up prompts correcting or improving the output ("the nav needs aria-label", "use article not div for the skill cards")
- Questions about specific lines you didn't understand
- The final version of each HTML file as it was committed

## What makes a good build chat

- Opens with a specific, detailed brief — not "make me a portfolio"
- Shows iteration — the first response is rarely the final one
- Includes questions about things you didn't understand
- The code you committed should be traceable back to this conversation

## Opening prompt example

> "I need to build the HTML structure for four pages: index.html (home), about.html, projects.html, and quotes.html. Requirements: semantic HTML5 elements throughout (header, nav, main, footer, section, article — no div soup), navigation linking all four pages consistently, no inline styles, no JavaScript yet. My name is Alex Rivera and I'm studying Business & Web Technologies at Curtin. Start with index.html."
